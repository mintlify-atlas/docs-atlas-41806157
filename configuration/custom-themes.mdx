---
title: "Creating Custom Color Themes for DankMaterialShell"
description: "How to define theme JSON files with Material Design 3 colors and load them into DMS, including dark and light variants and live reload."
---

Custom themes let you define your own Material Design 3 color palette that DankMaterialShell applies across every panel, widget, and overlay. A theme is a plain JSON file you create anywhere on disk, then point DMS at through the settings UI or `settings.json`. The shell reads the file at startup and watches it for changes while it is the active theme.

## Theme JSON structure

A theme file can define separate `dark` and `light` variants, or a single top-level palette used for both modes.

**Dual-variant theme (recommended):**

```json
{
  "dark": {
    "name": "My Theme Dark",
    "primary": "#00FFCC",
    ...
  },
  "light": {
    "name": "My Theme Light",
    "primary": "#00B899",
    ...
  }
}
```

**Single-variant theme (same palette for dark and light):**

```json
{
  "name": "My Theme",
  "primary": "#eeeeee",
  ...
}
```

## Required core colors

Every theme must include the following color keys. All values are hex color strings (e.g., `"#RRGGBB"`).

<AccordionGroup>
  <Accordion title="Primary colors">
    Colors used for buttons, active states, and primary interactive elements.

    | Key | Description |
    |-----|-------------|
    | `primary` | Main accent color for buttons, highlights, and active states |
    | `primaryText` | Text color that contrasts well against a `primary` background |
    | `primaryContainer` | A darker or lighter variant of `primary` used for containers |
  </Accordion>
  <Accordion title="Secondary and tint colors">
    Supporting accent colors that provide visual hierarchy and variety.

    | Key | Description |
    |-----|-------------|
    | `secondary` | Supporting accent color for secondary interactive elements |
    | `surfaceTint` | Tint color applied to surfaces; usually matches or derives from `primary` |
  </Accordion>
  <Accordion title="Surface colors">
    Colors for cards, panels, and layered UI surfaces.

    | Key | Description |
    |-----|-------------|
    | `surface` | Default surface color for cards and panels |
    | `surfaceText` | Primary text color rendered on `surface` backgrounds |
    | `surfaceVariant` | Alternate surface color for subtle visual differentiation |
    | `surfaceVariantText` | Text color for `surfaceVariant` backgrounds |
    | `surfaceContainer` | Container surface color, slightly different from `surface` |
    | `surfaceContainerHigh` | Elevated container color for layered interfaces |
    | `surfaceContainerHighest` | Highest-elevation container color for top-level surfaces |
  </Accordion>
  <Accordion title="Background colors">
    Colors for the overall application backdrop.

    | Key | Description |
    |-----|-------------|
    | `background` | Main background color for the entire interface |
    | `backgroundText` | Text color rendered directly on `background` |
  </Accordion>
  <Accordion title="Outline colors">
    Colors for borders and dividers.

    | Key | Description |
    |-----|-------------|
    | `outline` | Border and divider color used for subtle boundaries between elements |
  </Accordion>
</AccordionGroup>

## Optional semantic colors

You can also specify these optional keys. If omitted, DMS uses sensible defaults.

```json
{
  "error": "#f44336",
  "warning": "#ff9800",
  "info": "#2196f3"
}
```

| Key | Description |
|-----|-------------|
| `error` | Error states, delete buttons, and critical warnings |
| `warning` | Warning states and caution indicators |
| `info` | Informational states and neutral indicators |

## Matugen color scheme type

The optional `matugen_type` key controls the algorithm matugen uses when generating color tokens for GTK, terminals, and other apps from your wallpaper. If omitted, `scheme-tonal-spot` is used.

```json
{
  "matugen_type": "scheme-expressive"
}
```

| Value | Description |
|-------|-------------|
| `scheme-content` | Content-based color extraction |
| `scheme-expressive` | Expressive, vibrant color schemes |
| `scheme-fidelity` | High fidelity to source material |
| `scheme-fruit-salad` | Colorful, fruit salad-like schemes |
| `scheme-monochrome` | Monochromatic color schemes |
| `scheme-neutral` | Neutral, subdued color schemes |
| `scheme-rainbow` | Rainbow-like color schemes |
| `scheme-tonal-spot` | Tonal spot color schemes (default) |

## Loading your theme

<Steps>
  <Step title="Create your theme file">
    Save your theme JSON to any location, for example:

    ```
    ~/.config/DankMaterialShell/themes/mytheme.json
    ```
  </Step>
  <Step title="Select it in settings">
    Open Settings → **Appearance** → **Theme & Colors**, choose **Custom**, and enter the path to your file. Alternatively, edit `settings.json` directly:

    ```json
    {
      "currentThemeName": "custom",
      "customThemeFile": "/home/user/.config/DankMaterialShell/themes/mytheme.json"
    }
    ```
  </Step>
  <Step title="Iterate with live reload">
    With your custom theme active, save any change to the theme file and the shell updates immediately — no restart required.
  </Step>
</Steps>

<Note>
  Live reload only applies when the custom theme is the currently active theme. Edits to other theme files on disk are not automatically applied.
</Note>
