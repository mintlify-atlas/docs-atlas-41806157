title: "DankBar: Customizable Status Bar and Widgets"
description: "DankBar is a per-monitor status bar with workspace switching, system indicators, media controls, plugin slots, and a configurable left-center-right layout."
---

DankBar is the status bar component of DankMaterialShell. It runs as a per-monitor panel with a three-section layout — left, center, and right — and hosts a set of built-in widgets alongside any plugin widgets you install. You can configure which widgets appear in which section, toggle the bar from keybinds, and extend it with the plugin system.

## Bar positions

DankBar can be positioned at the top, bottom, left, or right edge of each screen. Position is set in **Settings → Appearance → DankBar Layout**. Each monitor gets its own bar instance.

## Plugin slots

DankBar has three layout sections. Each section is a configurable list of widget IDs:

- **Left section** — typically workspace switcher, launcher button, or running apps
- **Center section** — typically clock, media, or focused app name
- **Right section** — typically system indicators, audio, battery, tray

Plugin widgets are placed alongside built-in widgets using the same widget ID system.

## Layout configuration

The bar layout is configured in `~/.config/DankMaterialShell/settings.json` under the `dankBar*` keys, or edited interactively in **Settings → Appearance → DankBar Layout**.

```json
{
    "dankBarLeftWidgets": [
        {"widgetId": "workspaceSwitcher", "enabled": true},
        {"widgetId": "launcherButton", "enabled": true}
    ],
    "dankBarCenterWidgets": [
        {"widgetId": "clock", "enabled": true},
        {"widgetId": "media", "enabled": true}
    ],
    "dankBarRightWidgets": [
        {"widgetId": "audio", "enabled": true},
        {"widgetId": "battery", "enabled": true},
        {"widgetId": "systemTray", "enabled": true},
        {"widgetId": "myPlugin", "enabled": true}
    ]
}
```

Each entry specifies the `widgetId` (matching the built-in or plugin ID) and an `enabled` flag. Disabled widgets are hidden but remain in the list so you can re-enable them without reconfiguring the order.

## Multi-monitor support

Each connected monitor receives its own bar instance. Per-monitor instances are independent: the focused monitor's bar is used for IPC operations that target the active screen.

## Controlling the bar via IPC

<Tip>
  Bind bar visibility commands to a keybind in your compositor config to create an auto-hide or toggle-on-demand workflow.
</Tip>

```bash
# Toggle bar visibility
dms ipc call bar toggle

# Show or hide explicitly
dms ipc call bar reveal
dms ipc call bar hide

# Query current visibility
dms ipc call bar status
```

**Example Hyprland keybind:**

```conf
bind = SUPER, B, exec, qs -c dms ipc call bar toggle
```

**Example niri keybind:**

```kdl
binds {
    Mod+B { spawn "qs" "-c" "dms" "ipc" "call" "bar" "toggle"; }
}
```

## The Dock

Alongside the bar, DMS includes an application dock (`Dock` module) that shows pinned apps and running application windows. The dock supports:

- Per-app context menus
- Window management (focus, close, move)
- Overflow button for additional running apps
- A launcher button for quick access to the Spotlight launcher

The dock can be positioned independently of the bar and enabled or disabled in **Settings → Appearance**.
